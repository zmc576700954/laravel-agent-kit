# Profile：Filament Admin 迁移（示例）

本 Profile 只包含“后台 Admin 迁移到 Filament”场景的特有不变量与协作协议；不包含一次性任务剧本与临时 prompt。默认不启用，必须显式启用或满足启用条件才生效。

## 何时启用

- 任务属于后台 Admin 迁移（例如 OwlAdmin → Filament）
- 修改 `app/Filament/**` 或与 Filament 资源/页面/组件注册相关的代码
- 任务需要读写“迁移任务池/热文档”，并且用户已明确授权本会话写入

## Hard（场景不变量）

### 1) 边界与分层

- 后台迁移默认只动 Admin，不自动扩散到 API。
- 如项目已有 API 分层约定，保持现状，不在迁移任务中顺手改造分层。
- 未经明确要求，禁止修改数据库结构、接口契约、权限真相源、既有数据口径与用户可见行为。

### 2) 目录与实现约定

- Filament 页面/资源/RelationManagers/Widgets 放在 `app/Filament/**`。
- 旧后台目录（如 `app/Admin/**`）只做必要维护；后台新增能力默认使用 Filament。
- 涉及 `app/Filament/**` 改动时，禁止使用 `Filament\Tables\Actions\*`；统一使用 `Filament\Actions\*`。
- 涉及表单、表格、Infolist、页面标题、导航、面包屑、动作文案时，优先参考同资源或相邻资源的既有写法。

### 3) 热文档读取与串行写协议（如你的项目使用热文档）

- 仅在迁移批次任务中读取并维护用户指定的热文档；非迁移任务不要为了“会话启动”强行读取或更新。
- 热文档属于串行写入资源：同一时刻只能由一个被明确授权的会话写入；未授权会话默认只读，不得推进关键字段。

### 4) 迁移任务池真相源（如你的项目使用任务号/任务池）

- 迁移任务号/任务池的“唯一真相源”必须由用户指定（例如一个 master tasklist 文件中的 `NEXT_TASK_NO`）。
- 禁止通过扫描旧日志或历史目录推断最大任务号。
- 只有“任务在主索引登记且产生真实代码 diff”才占用迁移任务号；no-diff/skip/仅检查/普通文档修订不占用。

## 最小验证口径

- PHP 文件改动后，至少对变更文件执行：`php -l <file>`
- 涉及路由、Filament 页面访问路径或导航注册时：`php artisan route:list --path=<keyword>`
