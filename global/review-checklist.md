# Laravel 质量与安全 Review Checklist（Agent 自检/PR 自检）

## 0. 治理与范围（Reality Layer）

- [ ] 已明确本轮 In Scope / Out of Scope（用户指定优先）
- [ ] 已给出任务分级（轻/中/重）并说明触发依据
- [ ] 已给出计划/实施/验证闭环输出（Scope / Task Level / Plan / Implement / Verify）
- [ ] 未扫描整个项目，仅读取必要文件与少量同类参考
- [ ] 改动遵循最小改动原则（不顺手重构/清理/抽象升级）
- [ ] 分级匹配记录强度（如需任务清单/回退方案/执行证据）
- [ ] 若触发 Stop Conditions（迁移/权限/API 契约/中文文件/依赖/生产配置等）已先停下确认
- [ ] 如涉及迁移：未执行整体 migrate/rollback，采用单迁移文件定向执行策略
- [ ] 如涉及中文正文文件：保持编码/BOM/换行，避免高风险写入方式并复检关键段落

## 1. Context Discovery（基础信息）

- [ ] 已确认 Laravel/PHP 版本与关键依赖（来自 `composer.json` 或代码证据）
- [ ] 已确认测试框架（Pest/PHPUnit）与运行命令
- [ ] 已确认格式化/静态分析工具（Pint/PHPStan/Psalm/PHPCS）与运行命令

## 2. 路由与 HTTP 层

- [ ] 路由定义符合项目惯例（`routes/web.php`/`api.php`/分文件），命名与前缀一致
- [ ] Controller 保持薄：只做协调，不承载复杂业务
- [ ] 请求校验集中（Form Request 或项目现有方式），失败响应一致（422/错误结构）
- [ ] 错误处理符合项目风格（异常/错误码/统一响应）

## 3. 鉴权与安全

- [ ] 所有敏感操作有明确鉴权点（Policy/Gate/中间件/权限检查）
- [ ] 不依赖前端隐藏按钮实现安全
- [ ] 不输出敏感信息（token/密钥/用户隐私字段），日志不包含敏感 payload
- [ ] 防注入：避免字符串拼接 SQL，使用绑定参数/Eloquent
- [ ] 文件上传/下载：类型、大小、存储路径、访问权限、可执行文件风险已处理（如适用）

## 4. 数据库与迁移

- [ ] 任何 schema 变更都有 migration，且包含 `down()` 回滚
- [ ] 关键字段有索引/唯一约束/外键约束（按业务需求）
- [ ] 数据迁移（data migration）与 schema migration 区分清楚，必要时用事务/分批
- [ ] 时间字段、软删除、ID 策略（int/uuid/ulid）与项目一致

## 5. Eloquent 与性能

- [ ] 关系方法名与实际模型一致（不靠猜测）
- [ ] 避免 N+1：列表/聚合等场景已考虑 eager loading
- [ ] 大批量操作避免逐条保存（必要时用 chunk/batch）
- [ ] 缓存/队列/事件策略变更已明确影响与回滚（如适用）

## 6. 表单/Blade（如适用）

- [ ] Blade 输出做了转义（默认 `{{ }}`），避免 XSS
- [ ] 表单 CSRF 与方法伪造正确（`@csrf`/`@method`）
- [ ] 表单状态使用框架指令/项目惯例（如 `@checked` / `@selected`）（如适用且项目支持）

## 7. API（如适用）

- [ ] API 响应结构稳定，使用 Resource（或项目现有封装）保持一致
- [ ] HTTP 状态码正确（200/201/204/401/403/404/422）
- [ ] 路由模型绑定、分页、过滤与排序符合项目约定

## 8. 测试与验证（交付必选）

- [ ] 新功能/修复包含自动化测试（Feature 为主；业务逻辑可补 Unit）
- [ ] 测试覆盖关键分支：成功/失败/鉴权失败/校验失败
- [ ] 给出可执行验证命令（至少 test + format；静态分析若存在）
